# pr

Create a pull request for the current changes.

## Steps

0. **Проверка gh**  
   Выполни `gh auth status`. Если `gh` не установлен или не авторизован — сообщи пользователю до начала работы.

1. **Просмотр изменений**  
   Посмотри staged и unstaged изменения через `git status` и `git diff` (включая `git diff --staged`). Пойми, что именно изменилось.

2. **Сообщение коммита**  
   Напиши короткое и ясное commit message на основе изменений (например, в стиле conventional commits: `feat:`, `fix:`, `docs:`, `refactor:` и т.д.).

3. **Коммит и пуш**  
   Добавь все нужные файлы (`git add`), сделай коммит с этим сообщением и запушь в текущую ветку (`git push -u origin <branch>` или `git push`).

4. **Создание PR**  
   Для `gh pr create` — см. [.cursor/templates/gh-commands.md](.cursor/templates/gh-commands.md). Запиши title и body в файлы, вызови команду, удали временные файлы.

5. **Результат**  
   Верни пользователю URL созданного pull request'а.

## Заметки

- Если есть незакоммиченные изменения — спроси подтверждение перед коммитом или опиши, что будешь коммитить.
- Если `gh` не установлен или не залогинен — сообщи об этом и подскажи, как установить/авторизоваться.
- Если текущая ветка уже запушена и PR для неё уже есть — можно просто вернуть ссылку на существующий PR.
- **Windows + кириллица:** всегда используй запись в файлы для title и body PR.
